# Folio вЂ” React Native App Plan

## Overview

A native Android (and eventually iOS) app that does everything the desktop
version does вЂ” parse CAS PDFs, compute XIRR, suggest tax harvesting вЂ” but runs
fully on-device with no Python backend.

## Phase 1: Port the data layer (backend logic в†’ TypeScript)

The Python backend is ~1200 lines of well-isolated pure functions.
Each module ports independently and can be unit-tested in isolation.

### 1a. Models (`backend/app/models.py` в†’ `src/models/`)

Straightforward TypeScript interfaces/types:
- `Holding`, `Lot`, `TradeRow`, `Transaction`
- `PortfolioSummary`, `HoldingView`, `TaxHarvestReport`
- Enums: `AssetType`, `CostBasisType`

No logic вЂ” just type definitions.

### 1b. XIRR (`backend/app/services/xirr.py` в†’ `src/services/xirr.ts`)

Newton-method root-finding. Pure math вЂ” ports line-for-line.
- Input: array of `[date, amount]` cash flows
- Output: annualized XIRR as a decimal
- Same test vectors from `tests/test_core.py`

### 1c. Tax harvest (`backend/app/services/tax_harvest.py` в†’ `src/services/taxHarvest.ts`)

LTCG/STCG lot classification, FIFO matching, harvest suggestions.
- `classify_term(buyDate, sellDate)` в†’ LTCG | STCG
- FIFO matching of sells against buys
- Exemption-capped gain harvesting (в‚№1.25L)
- Loss harvesting logic

No external dependencies вЂ” pure functions over arrays of lots.

### 1d. Prices (`backend/app/services/prices.py` в†’ `src/services/prices.ts`)

- AMFI NAV fetch: `https://portal.amfiindia.com/spages/NAVAll.txt`
  в†’ Parse semicolon-delimited text, cache with TTL
- Stock prices: yfinance replacement needed
  - Option A: Use Yahoo Finance's unofficial API (`yf` npm package)
  - Option B: Use a free tier like Alpha Vantage / Twelve Data
  - Option C: NS EOD API (if available)
  - **Recommendation:** Try `yahoo-finance2` npm package first

### 1e. CAS PDF parser (`backend/app/parsers/cams_kfin_cas.py` в†’ `src/parsers/casPdf.ts`)

The hardest port. The Python parser uses PyMuPDF which extracts table cells
line-by-line. On React Native:
- `react-native-pdf` can render PDFs but doesn't extract text easily
- **Option A:** Use `react-native-pdf-extractor` (community, may be unstable)
- **Option B:** Use a WebView-based PDF.js approach
- **Option C:** Use a native PDF text extraction module
- **Fallback:** Upload parsed JSON directly instead of raw PDF
  (parse once on desktop, transfer the JSON data to mobile)

**Recommendation:** Build the parser to accept both a raw PDF AND a pre-parsed
JSON file (generated by the desktop app). This way users can parse on desktop
and transfer to mobile, while we work on native PDF extraction.

### 1f. Tradebook CSVs (`backend/app/parsers/zerodha.py` в†’ `src/parsers/zerodha.ts`)

CSV parsing is simpler вЂ” use `papaparse` npm package (already handles all edge cases).

## Phase 3: State management & persistence

Desktop uses `useState` + `localStorage`. React Native equivalent:

```

  в””в†’ cas-portfolio-v1 (same JSON structure)
```

State management can stay as `useState` + context for simplicity вЂ”
no need for Redux at this scale.
